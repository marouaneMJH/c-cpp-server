Project Details

Technical information and implementation details of C-Server

Architecture Overview

C-Server follows a modular architecture with several key components:

HTTP Parser

Responsible for parsing incoming HTTP requests, extracting method, headers, and body data.

Routing System

Maps URLs to handler functions, supporting different HTTP methods like GET and POST.

Response Generator

Creates HTTP responses with appropriate headers and content.

Socket Server

Handles TCP connections and request/response cycles.

Code Structure

The project is organized into several directories:

c-server/
├── include/         # Header files
│   ├── common/      # Common utilities
│   ├── http/        # HTTP-related components
│   └── server/      # Server configuration
├── src/             # Source code
│   ├── http/        # HTTP implementation
│   └── server/      # Server implementation
├── test/            # Test files
├── build/           # Compiled binaries
└── www/             # Web content

Key Features

  • Lightweight Core: Less than 2000 lines of C code for the core functionality
  • HTTP/1.1 Support: Basic support for the HTTP/1.1 protocol
  • Routing System: Register handlers for different URL paths
  • Static File Serving: Serve files from a directory
  • Content Type Detection: Automatically set appropriate content types
  • Error Handling: Proper handling of HTTP errors

Example Usage

Here's how you can define routes in your application:

// Define a simple route handler for GET /hello
ROUTE_GET("/hello", {
    char *response = "Hello, World!";
    send_response(client_fd, 200, "text/plain", response, strlen(response));
})

// Define a route handler for POST /echo
ROUTE_POST("/echo", {

Performance

C-Server is designed to be efficient and lightweight:

  • Memory usage: Less than 5MB for the server process
  • Can handle hundreds of concurrent connections
  • Fast startup time (less than 50ms)
  • Low latency response time (typically < 1ms for static content)

Future Roadmap

  • HTTP/2 support
  • WebSocket protocol integration
  • Configuration file support
  • Improved logging and metrics
  • HTTPS/TLS support
